Class: Conditioned

lib/conditioned~ Conditioned

This class is a mixin which adds the glue needed for an object to allow other objects to know when certain conditions have been met. This mixin depends on an event emitting and listening framework of the sort provided by simple_event_emitter. The object using this mixin does not have to use simple_event_emitter but must offer the same interface.

Since this mixin builds on simple_event_emitter or something similar, the following are true:

  • a condition name is an event name; setting the condition triggers the event.
  • a condition listener is an event listener, and should have the same signature: listener(ev) where ev is an object related to the event.

Events and conditions live in the same namespace. Consequently, classes using this mixin should take care of avoiding mixing condition and event names. For instance, if a class emits an event using a condition name, this may result in condition listeners being executed, but the condition won't be recorded as being in effect as would be the case if _setCondition would be called.

new Conditioned()

This constructor must be called by the class using this mixin.

getCondition(name) → {boolean}

This is a convenience method to check whether a condition has been reached or not. In most cases, asynchronous code should call whenCondition rather than poll repeatedly by using this method. However, there are some rare situations in which polling is called for.

whenCondition(name, listener)

This method is called so that a listener function is executed when a certain condition is reached. If the condition has already been reached, then the listener will be executed immediately. If not, then the listener will be executed as soon as the condition is reached. In any case, the listener is executed once and only once per call to this method.

Parameters:
Name Type Description
name string

The name of the condition that the caller is interested in.
listener function

The listener to execute when the condition is reached.
